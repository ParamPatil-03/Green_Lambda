01 — The Problem

Energy Remains
Invisible to Developers

Serverless computing has become a dominant cloud paradigm due to its scalability and ease of deployment. However, current profiling tools focus on execution time, latency, and monetary cost — while energy consumption remains entirely opaque.

As cloud infrastructures scale, the cumulative energy footprint of serverless functions contributes significantly to operational costs and environmental impact. Developers lack mechanisms to estimate energy usage before deployment.

03 — The Framework

From Code to Carbon

Our approach combines static analysis, execution profiling, and ML prediction in a four-phase discipline to deliver pre-deployment energy estimates.

01

Extract

Parse serverless function code to extract static features — cyclomatic complexity, I/O operations, memory allocation patterns, loop depth, and external API calls.

02

Profile

Collect historical execution data — invocation frequency, runtime duration, cold start overhead, CPU/memory utilization, and network throughput as dynamic features.

03

Predict

Feed combined feature vectors into trained ML models (XGBoost, Random Forest, Neural Network) to predict energy consumption in Wh per invocation.

04

Optimize

Generate actionable insights — SHAP-based feature importance, code pattern analysis, and concrete optimization recommendations to reduce energy footprint.
